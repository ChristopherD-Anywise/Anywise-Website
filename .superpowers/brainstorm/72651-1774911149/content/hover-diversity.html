Hover Effect Diversity

Currently almost every interactive element does the same thing: translateY(-6px) + green box-shadow. Here's a plan to give each section its own interaction personality.

Capability Cards
Keep Current (Already Distinctive)
translateY(-6px) + cursor spotlight + gradient top line
=
No change — the cursor-tracking spotlight and gradient backgrounds make these unique enough
Product Cards
Keep 3D Tilt, Remove Lift
3D tilt + translateY(-6px) + image zoom + green overlay
→
3D tilt only + subtle image zoom. Remove the -6px lift (tilt already creates depth). Remove green image overlay (let the image speak).
The tilt is the star here — the lift competes with it. Cleaner to let one effect do the work.
News Cards
Subtle Scale Instead of Lift
translateY(-6px) + green box-shadow + image zoom + green overlay + date color change + arrow reveal
→
scale(1.02) on whole card + image zoom. Remove green overlay, remove green shadow. Keep the arrow reveal — that's the good part.
News cards should feel clickable and light, not dramatic. A gentle scale says "I'm a link" without the heavy green glow.
Approach Features
Becoming Numbered List — Text Highlight Only
translateX(6px) + translateY(-2px) + left accent line + border color change
→
Number color shifts from #39a849 to #c3e01b on hover. Title text brightens. No movement.
As a numbered list, movement-based hover would feel odd. Colour shift is more appropriate for text content.
Track Record Stats
Becoming Inline Row — No Hover
translateY(-4px) + border change + gradient bottom line
→
No hover effect. Stats are informational, not interactive. Remove unnecessary interaction suggestion.
Not everything needs a hover state. Stats are display-only — adding hover implies they're clickable when they're not.
CTA Box
Remove Box Hover, Keep Button Hover
Whole box: border change + green shadow + image zoom
→
Keep subtle image zoom. Remove box border/shadow change — the buttons are the interactive elements, not the container.
Hovering the entire CTA section creates a confusing interaction target. Focus hover on the actual buttons.

→
N/A — values become inline text flow, no hover.
