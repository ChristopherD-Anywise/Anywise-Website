Implementation Approach

How should we architect the light/dark theming system?

A

CSS Custom Properties Swap — Recommended

Override all --bg-*, --text-*, --border-*, and --accent-* variables under a [data-theme="light"] selector on the root. JavaScript detects AEDT time and sets the attribute on page load.

Pros

  • Single source of truth — all colors flow from variables
  • Minimal CSS additions (~40 lines of variable overrides)
  • Globe texture swap handled separately in JS
  • Easy to extend later (manual toggle, user preference)
  • Smooth transition via transition: background-color 0.5s, color 0.5s on body

Cons

  • Some hardcoded rgba() values throughout CSS need individual overrides
  • Hero overlay gradients need separate light-mode rules
B

Duplicate Stylesheet Approach

Create a complete parallel set of CSS rules scoped under .light-mode class. Toggle the class on body based on time.

Pros

  • Full control over every element in light mode
  • No risk of missed variable references

Cons

  • Massive CSS duplication (~500+ lines)
  • Maintenance nightmare — every change needs updating in both themes
  • Bloats file size significantly
C

Hybrid: Variables + Targeted Overrides

Use CSS custom property swaps for the bulk of theming (backgrounds, text, borders), but write targeted [data-theme="light"] rules for complex components that use hardcoded values (hero overlay, capability cards, globe glow shader).

Pros

  • Best of both worlds — variables handle 80%, targeted rules handle edge cases
  • Handles hardcoded rgba() values and gradient overlays properly
  • Moderate CSS additions (~80-100 lines)
  • Clean separation between variable-driven and component-specific theming

Cons

  • Slightly more complex than pure variable swap
  • Need to audit all hardcoded color values

💡 Recommendation: Approach C (Hybrid)

The Anywise site has ~15 hardcoded rgba() values in gradients, overlays, and card effects that won't respond to variable swaps alone. The hybrid approach handles these cleanly while keeping the variable system as the primary mechanism. It's the right balance of maintainability and completeness.
